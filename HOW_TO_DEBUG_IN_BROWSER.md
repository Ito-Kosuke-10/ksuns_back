# ブラウザの開発者ツールでエラーを確認する方法

## ステップ1: Networkタブでリクエストを確認

### 1. 開発者ツールを開く
- **F12キー**を押す、または
- 右クリック → 「検証」を選択

### 2. Networkタブを開く
- 開発者ツールの上部タブから「**Network**」をクリック

### 3. リクエストを記録する準備
- Networkタブの左上に**赤い丸（●）**があることを確認（記録中であることを示す）
- もし灰色の丸（○）なら、クリックして記録を開始

### 4. ページをリロードまたはGoogleログインを再試行
- エラーが表示されているページで**Ctrl + R**（またはF5）でリロード
- または、Googleログインを再度試行

### 5. リクエストを確認
- Networkタブにリクエストの一覧が表示されます
- `/auth/google/callback` というリクエストを探す
- そのリクエストを**クリック**して選択

### 6. レスポンスの詳細を確認
- 右側のパネルで「**Response**」タブをクリック
- または「**Preview**」タブをクリック
- エラーメッセージの詳細が表示されます

## ステップ2: リクエストの詳細を確認

### 確認すべき情報

1. **ステータスコード**
   - リクエストの右側に表示される（例：`500`、`200`など）
   - `500` = Internal Server Error

2. **レスポンスボディ**
   - 「Response」タブまたは「Preview」タブで確認
   - エラーの詳細が表示されます
   - 例：`{"detail": "Internal server error", "error_type": "..."}`

3. **リクエストURL**
   - リクエストをクリックすると、URLが表示されます
   - クエリパラメータ（`?code=...&error=...`など）を確認

## ステップ3: Consoleタブでエラーを確認

1. 開発者ツールの「**Console**」タブをクリック
2. 赤いエラーメッセージがないか確認
3. エラーメッセージがあれば、それをコピー

## よくあるエラーパターン

### パターン1: `{"detail": "Internal server error"}`
- バックエンドで予期しないエラーが発生
- Azureのログを確認する必要がある

### パターン2: `{"detail": [{"type": "missing", "loc": ["query", "code"], ...}]}`
- `code`パラメータが欠けている
- Googleログインをキャンセルした場合に発生

### パターン3: `{"detail": "oauth_error_redirect_uri_mismatch"}`
- Google OAuthの設定エラー
- リダイレクトURIが一致していない

## Azureのログを確認する方法

ローカルのターミナルではなく、**Azure Portal**でログを確認する必要があります：

1. Azure Portalにログイン
2. App Service（バックエンド）を開く
3. 「**ログストリーム**」または「**ログ**」を選択
4. リアルタイムでログを確認

または、Azure Portalの「**高度なツール (Kudu)**」からログファイルを確認することもできます。
